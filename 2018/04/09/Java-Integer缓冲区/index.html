Integer缓冲区

针对-128和127之间的数据，Java做了一个数据缓冲区，如果数据属于这个范围内，会从缓冲区里面直接去数据，并不会创建新的空间

使用Integer声明的变量是一个对象，不再是基本数据类型，例如 Integer integer1 = 128，是将128进行装箱操作，使用Integer.valueOf(128)进行装箱。
由于Integer声明的变量是对象，因此我们会认为两个对象使用 == 进行比较时，肯定是false。

结果显示，integer3和integer4指向的是同一个地址！！！

查看Integer原码可知，针对「-128」至「127」之间的数据，Java做了一个数据缓冲区，如果数据属于这个范围内，会从缓冲区里面直接去数据，并不会创建新的空间。

数据缓冲区中取数据的情况仅限于使用『=』进行直接赋值操作，不包括使用new Integer()创建的对象，使用new Integer()创建的对象不会出现 == 判断为true的情况。
