单例

简单来说，就是整个程序生命周期内某个对象只会生成一次

整个程序生命周期内只能出现一个它的实例， 又或者，某些对象的创建非常复杂非常消耗资源，且它是不会随着程序的运行而有所改变的，那么我们也期望只生产一次这个对象，后面直接拿来复用！ 那么这种只会生成一次某个对象的编码技巧，我们就将其称为单例模式！

实现思路

一个类能返回对象一个引用(永远是同一个)和一个获得该实例的方法（必须是静态方法，通常使用getInstance这个名 称）；当我们调用这个方法时，如果类持有的引用不为空就返回这个引用，如果类保持的引用为空就创建该类的实例并将实例的引用赋予该类保持的引用；同时我们 还将该类的构造函数定义为私有方法，这样其他处的代码就无法通过调用该类的构造函数来实例化该类的对象，只有通过该类提供的静态方法来得到该类的唯一实例。

目前网络上认可较多的是5种实现，分别是
懒汉式、饿汉式、懒汉式-线程安全、内部类、枚举

下面分别介绍这几种

饿汉式

饿汉式对于一些比较耗时的对象来说是不合适的，如果这个对象根本就没用上还是会被实例化，这对于服务器资源来说是一种浪费。

懒汉式

懒汉式优点

  • 单例对象不会随着类的加载而实例化，只会等到有需要时才会去实例化一次。

缺点

  • 线程不安全(产生于if-else处)，如果多个线程同时调度下，A、B两个线程都进入了getInstance()中，此时instance为空，那么就会被创建2次。

由此引出了懒汉式的线程安全版本

实现了线程安全
双重if检查，且锁操作在最内一层if。最大程度减少线程等待时间。

懒汉式（内部类）

Inside是一个静态内部类，就算有再多的线程，它也只会被类加载器装载一次，而里面的instance会随着Inside的类装载而被实例化，所以，它绝对是唯一不重复的，这一点，它和饿汉式的效果是一样的。但和饿汉式不同的时，Inside类只有当我们调用Singleton类的getInstance方法时才会被类加载器加载，所以，Singleton的实例也只会在这个时候被创建，所以它不会造成资源浪费！
