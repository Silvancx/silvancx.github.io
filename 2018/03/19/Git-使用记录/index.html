Git 使用记录

『Git』是目前世界上最先进的分布式版本控制系统（没有之一）。
目前公司使用地版本管理还是SVN，之前遇到有个同事离职交接的项目文档，里面关于接口文档的版本处理太原始了。。。就像下面这样。

处理这样的文档是真心累。

集中式与分布式

参考『廖雪峰』的文章集中式与分布式

Git安装

Mac OSX下安装

一是安装homebrew，然后通过homebrew安装Git，具体方法请参考homebrew的文档：http://brew.sh/。

第二种方法更简单，也是推荐的方法，就是直接从AppStore安装Xcode，Xcode集成了Git，不过默认没有安装，你需要运行Xcode，选择菜单「Xcode」->「Preferences」，在弹出窗口中找到「Downloads」，选择「Command Line Tools」，点「Install」就可以完成安装了。

Git-常用命令

git init 在当前目录创建一个「Git Repository」，目录不要带有中文名，避免莫名其妙的问题。会在当前目录下生成一个隐藏目录.git。
git add 将文件添加到仓库。
git commit 将文件提交到仓库。
git commit -m "本次提交的描述信息"
git log 命令显示从最近到最远的提交日志历史
git reset --hard commit_id 版本回退
git reflog 查看命令历史
git status 查看仓库状态(哪些文件被修改了，哪些文件还未被添加到仓库)
git rm test.txt 删除文件(删除文件后需要git commit进行提交操作)
git checkout -- test.txt 用版本库里的版本替换工作区的版本
ssh-keygen -t rsa -C "youremail@example.com" 创建SSH Key
你需要把邮件地址换成你自己的邮件地址，然后一路回车，使用默认值即可，由于这个Key也不是用于军事目的，所以也无需设置密码。
如果一切顺利的话，可以在用户主目录里找到.ssh目录，里面有id_rsa和id_rsa.pub两个文件，这两个就是SSH Key的秘钥对，id_rsa是私钥，不能泄露出去，id_rsa.pub是公钥，可以放心地告诉任何人。
git clone 从远程仓库克隆(协议支持https、ssh(原生))
git checkout -b dev 创建并切换分支，相当于以下两条命令

git branch 列出当前所有分支当前分支前面会有一个*号。
git merge option 用于合并指定分支到当前分支。
git branch -d option 删除分支
git log --graph 查看分之合并图

参考文章：Git 教程、git-scm.com
